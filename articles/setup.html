Router setup

The main trick to enable client side routing is to configure the server side router to serve content when the request get a 404 HTTP status code. On GitHub Pages, you can achieve it creating a 404.html file.

You can start simply by coping your index.html into a 404.html file.

Of course you need also to enable GitHub Pages in your repository settings. You will notice there are two kinds of static websites you can create.

The first kind happens when your repository has a name like YOUR-ACCOUNT-OR-ORGANIZATION.github.io. In this case the homepage will be https://YOUR-DOMAIN/ where YOUR-DOMAIN will default to YOUR-ACCOUNT-OR-ORGANIZATION.github.io. You can also set your custom domain, check out the official GitHub Pages documentation.

The second kind is a website that will have an homepage like https://YOUR-DOMAIN/YOUR-REPO-NAME/. In this case it is required that you add a base tag in your HTML head, something like the following: 

<base href="/YOUR-REPO-NAME/" />

To simulate this behaviour locally, the development server reads the homepage attribute in the package.json
