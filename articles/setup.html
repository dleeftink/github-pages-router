Router setup

Generally speaking, the main trick to enable client side routing is to configure the server side router to serve content whenever the request gets a 404 HTTP status code. On GitHub Pages, you can achieve this by creating a 404.html file.

It is not required that both files have the same content but you can start simply coping your index.html into a 404.html file.

Of course you need also to enable GitHub Pages in your repository settings. You will notice there are two kinds of static websites you can create.
